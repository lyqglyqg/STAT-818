R Functions

  • An R function, like a mathematical function, maps each input argument to an output value.

  • Every R package contains at least one function. Always use implemented functions in a published R package to save time.

  • R users can define their own functions.

A user-defined R function has the following structure.

1
2
3
4
5
6

my_function <- function(arg1, arg2, ... ){

   function body
   
   return( outputs )
}

my_function: function name. Can contain alphabetics, underscore, numbers (cannot be at the 1st place), and period sign “.”. For example: “my_function”, “my_function.v1”.

arg1, arg2, ...: function argument(s). Can have default values.

function body: statements that are executed when the function runs.

return( outputs ): function outputs. Use return() to specify the list of values you want to output.

Examples

  • Here is an example R function for converting temperatures from Fahrenheit to Celsius:
1
2
3
4

fahrenheit.to.celsius <- function(temp.F) {
  temp.C <- (temp.F - 32) * 5 / 9 
  return(temp.C) 
}

After defining the function, you can call the function like below.

Variable scope

  • A variable that is visible only within function body is said to be local to that function. In fahrenheit.to.celsius() function, tmp.C is a local variable

  • Local variables disappear after the function is defined or called.

  • A global variable will not be changed if it was used in function as an argument.

Argument defauts in functions

One can specify a defacut value for an argument in a function. When calling the function, if no argument value is given, then the function will run by the default value of that argument.

1
2
3
4
5
6
7
8

fahrenheit.to.celsius <- function(temp.F=80) {
  temp.C <- (temp.F - 32) * 5 / 9 
  return(temp.C) 
}


> fahrenheit.to.celsius()
[1] 26.66667

Control statements

Control statements are very useful in writing R functions. Commonly used control statements in R include conditional statements and loop statement.

Conditional statement

The if else statement

The if else statement has the following structure.

1
2
3
4
5

if("condition is satisfied"){
    "do something"
} else {
    "otherwise do something else"
}
  • Let us write an example R function using the if else statement.

celsius.weather.report <- function(temp.C){
    if(temp.C<0){
        print("It is freezing outside. Drive carefully!")
    } else {
	print(paste("Current outside temperature is ", temp.C, "°C.", sep=""))
    }
}  ### Notice this function does not return an output value.

> celsius.weather.report(15)
[1] "Current outside temperature is 15°C."
>
> celsius.weather.report(-5)
[1] "It is freezing outside. Drive carefully!"

celsius.weather.report_morning <- function(temp.C.vector){  ### This function requires a length-5 input vector for hourly temperature from 7 to 11 am.
    for(i in 1:5){
       print(paste("The temperature at ", (i+6), "am is ", temp.C.vector[i], "°C.", sep=""))
    }
}
> 
> celsius.weather.report_morning(c(-2, 1, 3, 8, 10))

[1] "The temperature at 8am is 1°C."
[1] "The temperature at 9am is 3°C."
[1] "The temperature at 10am is 8°C."
[1] "The temperature at 11am is 10°C."

celsius.weather.freezing.alarm <- function(temp.C.vector){
   i <- 1
   while(temp.C.vector[i] > 0){
       print(paste("The current temperature is ", temp.C.vector[i], "°C.", sep="")) 
       i <- i+1
   }
   
   print("It is freezing now. Drive carefully!!")
}

Debugging

Debugging your R codes or functions is more of a skill of experience. It could be a cumbersome process sometime. The most useful trick I found for R debugging is to Google!!. Especially when you use a published R package. Most of time, you will find others talking about the same error massege if it is a common error. If you are lucky, you will find someone post online a solution to it.

For more tricks on R debugging, you can find a good tutorial here.
