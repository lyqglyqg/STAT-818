We use an R package named edgeR to learn how to do gene differential expression (DE) analysis using RNAseq count data.

Introduction

edgeR is a widely used R package for conducting DE analysis with RNAseq data. Its statistical analysis is based on negative binomial distribution. It provides Bayes estimation, generalized linear modelling, Fisher exact test, and Quasi-likelihood test.

Installation

First, you need to know your R version. To check that, use

R.Version()

If your R version is >3.5, then install edgeR use

if (!requireNamespace("BiocManager", quietly = TRUE))
    install.packages("BiocManager")

BiocManager::install("edgeR")

If your R version is <3.5, then install edgeR use

source("http://bioconductor.org/biocLite.R"ï¼‰
biocLite('edgeR')

edgeR vignette

> library(edgeR)
> edgeRUsersGuide()

Analysis step 1: read in data

The “geneCounts.csv” file contains RNAseq count data from 419 ovarian cancer patients from TCGA data and 88 normal controls from GTEX data.

X_c <- read.csv("geneCounts.csv", row.names=1, header=TRUE)

X_c[1:10, 1:10]

Creat a cancer indicator and a ‘DGEList’ object

First, creat a group indicator of cancer vs normal control, with 1 indicating ovarian cancer. If a rowname starts by “TCGA”, then it is RNAseq data from a cancer subject. If it starts by “GTEX”, it is a normal control.

sample_vec <- sapply(rownames(X_c), function(x) unlist(strsplit(x, "-"))[[1]][1]) 
group <- as.numeric(sample_vec == "TCGA")
table(group)

edgeR can create a ‘DGEList’ object that contains a table of counts (rows=features, columns=samples), group indicator, library size (optional) for the downstream DE analysis.

y <- DGEList(counts=t(X_c),group=group)

Filtering and normalization

edgeR can filter out low quality genes (genes with low counts) and can normalize the gene counts to scale the raw library sizes.

keep <- filterByExpr(y)  
y <- y[keep,,keep.lib.sizes=FALSE]
y <- calcNormFactors(y)
design <- model.matrix(~group)
y <- estimateDisp(y,design)

DE analysis

Use the glmQLFit and glmQLFTest to run the DE analysis and get a p-value for each gene.

fit <- glmQLFit(y,design)
qlf <- glmQLFTest(fit,coef=2)

Calculate adjusted p-values and sort the output table

We need to control for multiple testing here (this is a statistical concept not required for this course), the following syntax calculate the false discovery rates (fdr or adjusted p-values) and sort the output table from the most significant to least significant genes.

qlfT <- qlf$table
qlfT_sort <- qlfT[order(qlfT[,4]),]
qlfT_sort$fdr <- p.adjust(qlfT_sort[,4], method ="fdr")

qlfT_sort[1:10,]

The smaller the PValue or the fdr, the more differently expressed for the gene between the ovarian cancer subjects and the normal controls.

The logFC is for “log fold change” with base 2. “logFC=3.14” means the average count number of the ovarian cancer group is $2^{3.14}$ times of that if the normal control group for the gene.

Generate plots to visualize outputs

First, we generate a heatmap for the most 200 differentially expressed genes (similar to what you did in the midterm exam).

library(pheatmap)

topGene <- rownames(qlfT_sort[1:200,])
idtmp <- match(topGene, colnames(X_c))
heatmapGene <- X_c[ , idtmp]

sample_names_short <- c(paste("GTEX-", c(1:88), sep=""), paste("TCGA-", c(1:419), sep=""))
group_col <- c(rep("GTEX", 88), rep("TCGA", 419))
my_sample_col <- data.frame(sample = group_col) #sample_names_short)
row.names(my_sample_col) <- sample_names_short

row.names(heatmapGene) <- sample_names_short


pheatmap(t(heatmapGene), annotation_col = my_sample_col, fontsize=3, cluster_rows=F, cluster_cols =F)

Next, we generate a plot called “volvano plot”, which can show both the scales of “logFC” and the significance of “PValues” (in term of $-log_10(PValue)$). You will need to install the package “EnhancedVolcano.” (See https://bioconductor.org/packages/devel/bioc/vignettes/EnhancedVolcano/inst/doc/EnhancedVolcano.html for details).

if (!requireNamespace('BiocManager', quietly = TRUE))
    install.packages('BiocManager')

BiocManager::install('EnhancedVolcano')


library(EnhancedVolcano)

qlfT_sort_vol <- qlfT_sort

  EnhancedVolcano(qlfT_sort_vol,
    lab = rownames(qlfT_sort_vol),
    x = 'logFC',
    y = 'PValue')

One can specify more details to make the plots more informative. For example:

keyvals <- rep('gray', dim(qlfT_sort_vol)[1])
keyvals[101:500] <- 'royalblue'
keyvals[1:100] <-  'gold'

names(keyvals)[keyvals == 'gold'] <- 'Strong gene'
names(keyvals)[keyvals == 'royalblue'] <- 'Weak gene'
names(keyvals)[keyvals == 'gray'] <- 'No significant gene'

shreshold <- 6.14e-9


  EnhancedVolcano(qlfT_sort_vol,
    lab = rownames(qlfT_sort_vol),
    x = 'logFC',
    y = 'PValue',
    selectLab = rownames(qlfT_sort_vol)[which(names(keyvals) %in% c('Strong gene', 'Weak gene'))],
    xlab = bquote(~Log[2]~ 'fold change'),
    title = NULL,
    pCutoff = 1.0e-55,
    FCcutoff = 2.0,
    pointSize = 3.0,
    labSize = 3.0,
    colCustom = keyvals,

    hline = c(shreshold),
    hlineCol = c('red'))
